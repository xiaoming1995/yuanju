# 合盘页生辰输入体验对齐设计

## 背景

当前合盘页 [CompatibilityPage.tsx](/Users/liujiming/web/yuanju/frontend/src/pages/CompatibilityPage.tsx) 使用数字输入框录入年月日时，和单人测算页 [HomePage.tsx](/Users/liujiming/web/yuanju/frontend/src/pages/HomePage.tsx) 的下拉选择、时辰选择、历法联动体验不一致。用户在合盘场景下需要连续录入两个人的信息，这种差异会放大输入摩擦。

## 目标

- 让合盘页与单人测算页使用同一套出生信息交互模型
- 消除年月日时自由数字输入带来的别扭感和无效输入
- 将日期、历法、闰月、时辰逻辑收敛为一套可复用实现

## 方案

新增一个前端共享输入段组件，承载以下能力：

- 性别切换按钮
- 公历 / 农历切换按钮
- 年、月、日下拉选择
- 农历闰月联动
- 十二时辰选择器
- 年月变化时自动修正可选日期范围

页面接入方式：

- `HomePage` 改为复用该共享输入段
- `CompatibilityPage` 复用该共享输入段两次，分别录入“我的生辰”和“对方的生辰”
- 合盘页保留现有提交与跳转逻辑，不改 API

## 边界

本次不包含：

- 省份 / 经度 / 真太阳时修正
- 早子时额外开关
- 合盘算法、结果页、历史页修改
- 新的分步向导或页面结构重做

## 验证

- 单人测算页仍可正常提交
- 合盘页可正常录入双方生日并提交
- 公历 / 农历切换、闰月切换、日期联动表现一致
- 前端构建通过
