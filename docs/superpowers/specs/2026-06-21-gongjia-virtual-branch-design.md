# 原局夹拱虚支算法设计

## 背景

当前八字引擎已输出四柱、五行、十神、藏干、神煞、用神、命格、大运与流年事件信号，但没有识别“夹拱”。

本设计新增“原局夹支”算法：当原局相邻两柱天干相同，且两柱地支在十二地支顺序中正好隔一个字时，中间地支作为暗藏虚支进入信号层。夹支用于表达“意料之外、情理之中”的暗线，不作为第五柱。

## 目标

- 识别原局相邻两柱形成的夹支。
- 将夹支作为结构化字段返回给前端。
- 为夹支计算藏干、相对日主十神，以及可按地支引用的神煞。
- 让大运、流年可以引动既有夹支，输出中性事件信号。
- 保持现有五行、身强弱、用神、调候、命格结果稳定。

## 非目标

- 不让大运或流年参与生成夹支。
- 不把夹支当作真实第五柱。
- 不用夹支改写五行统计、身强弱、用神、忌神、调候、命格。
- 不在第一版对夹支事件直接判吉凶。
- 不扩展合盘、性格画像或付费报告的主算法，除非后续单独设计。

## 术语

- 真实四柱：年柱、月柱、日柱、时柱。
- 相邻两柱：年柱-月柱、月柱-日柱、日柱-时柱。
- 夹支：相邻两柱天干相同，地支隔一位时，中间被夹出的地支。
- 虚支：夹支的性质。它可参与信号解释，但不等同真实地支落柱。
- 引动：大运或流年与夹支发生冲、合、刑、害、三合、三会关系。

## 夹支识别规则

系统只检查原局相邻两柱：

```text
年柱 - 月柱
月柱 - 日柱
日柱 - 时柱
```

成立条件：

```text
1. 两柱天干相同
2. 两柱地支在十二支环形顺序中正好隔一个字
3. 中间地支为夹支
```

十二支顺序：

```text
子 丑 寅 卯 辰 巳 午 未 申 酉 戌 亥
```

可识别的地支对：

| 地支对 | 夹支 |
| --- | --- |
| 子-寅 / 寅-子 | 丑 |
| 丑-卯 / 卯-丑 | 寅 |
| 寅-辰 / 辰-寅 | 卯 |
| 卯-巳 / 巳-卯 | 辰 |
| 辰-午 / 午-辰 | 巳 |
| 巳-未 / 未-巳 | 午 |
| 午-申 / 申-午 | 未 |
| 未-酉 / 酉-未 | 申 |
| 申-戌 / 戌-申 | 酉 |
| 酉-亥 / 亥-酉 | 戌 |
| 戌-子 / 子-戌 | 亥 |
| 亥-丑 / 丑-亥 | 子 |

示例：

```text
甲子 年柱 + 甲寅 月柱
子 - 丑 - 寅
=> 年月夹丑
```

反向同样成立：

```text
甲寅 年柱 + 甲子 月柱
寅 - 丑 - 子
=> 年月夹丑
```

不成立示例：

```text
甲子 + 乙寅：天干不同
甲子 + 甲卯：地支不是隔一位
年柱 + 日柱：非相邻柱，第一版不检查
原局 + 大运：大运不参与生成夹支
原局 + 流年：流年不参与生成夹支
```

## 数据结构

后端在 `BaziResult` 增加可选数组：

```json
{
  "gong_jia": [
    {
      "source": "year_month",
      "source_labels": ["年柱", "月柱"],
      "same_gan": "甲",
      "source_zhis": ["子", "寅"],
      "virtual_zhi": "丑",
      "hide_gan": ["己", "癸", "辛"],
      "shishen": ["正财", "正印", "正官"],
      "shensha": ["天乙贵人"],
      "meaning": "原局年柱与月柱天干同为甲，地支子寅夹丑，丑为暗藏虚支。"
    }
  ]
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `source` | `year_month`、`month_day`、`day_hour` |
| `source_labels` | 前端展示用中文柱位 |
| `same_gan` | 两柱相同天干 |
| `source_zhis` | 两个真实地支，保留原柱位顺序 |
| `virtual_zhi` | 夹出的虚支 |
| `hide_gan` | 虚支藏干，复用现有藏干表 |
| `shishen` | 虚支藏干相对日主的十神 |
| `shensha` | 虚支按地支引用到的神煞 |
| `meaning` | 可直接展示的简短解释 |

如果没有夹支，`gong_jia` 为空或省略。前端不得因旧命盘快照缺少该字段而报错。

## 神煞口径

夹支只按“虚支地支”引用神煞，不构造成完整虚柱。

可计算：

```text
以年干、日干、年支、日支、月支等为基准，只检查某个地支是否命中。
例如：天乙贵人、桃花、驿马、华盖、将星、劫煞、灾煞等。
```

不计算：

```text
必须依赖“该柱自身天干 + 该柱自身地支”的神煞。
虚支没有自己的天干，不应强行拼出类似“甲丑”的虚柱。
```

展示文案应表达为：

```text
拱天乙贵人：夹出丑，丑为日干或年干所取贵人地，主暗中贵人、意外相助。
```

不得表达为：

```text
甲丑柱带天乙贵人。
```

## 大运与流年引动

大运、流年不能生成夹支，但可以引动已经存在的原局夹支。

事件检测集合：

```text
真实四支：年支、月支、日支、时支
虚支：gong_jia[].virtual_zhi
大运支
流年支
```

触发类型：

| 触发 | 输出方向 |
| --- | --- |
| 大运或流年冲夹支 | 暗藏虚支被冲动 |
| 大运或流年合夹支 | 暗藏虚支被合动 |
| 大运或流年刑夹支 | 暗藏虚支受刑 |
| 大运或流年害夹支 | 暗藏虚支受害 |
| 大运或流年与真实支、夹支补足三合 | 夹支参与三合成局 |
| 大运或流年与真实支、夹支补足三会 | 夹支参与三会成局 |

第一版所有夹支事件：

```text
Polarity = 中性
Source = 夹拱
```

不直接判吉凶。吉凶由报告层结合夹支十神、神煞、性别、大运流年上下文解释。

事件证据示例：

```text
流年未冲原局年月夹支丑，暗藏虚支被冲动，主意料之外但有迹可循的人事变化。
```

```text
大运子合原局年月夹支丑，夹支被合动，暗线关系、资源或贵人信号更容易显化。
```

```text
流年酉与原局巳、夹支丑补足巳酉丑金局，夹支参与成局，暗藏力量被引出。
```

## 前端展示

结果页在基本排盘或专业信息附近新增“原局夹拱”模块。无夹支时不显示模块。

展示内容：

```text
原局夹拱
年柱甲子 与 月柱甲寅：夹出丑
藏干：己 / 癸 / 辛
十神：正财 / 正印 / 正官
神煞：天乙贵人
提示：夹出之支为暗藏信号，不改原局五行与用神，仅参与应期引动。
```

展示原则：

- 明确标注“虚支”或“暗藏信号”。
- 不把夹支渲染成第五柱。
- 神煞可复用现有神煞徽章和说明弹层，但文案需说明是“拱神煞”。
- 旧命盘缺少 `gong_jia` 时正常降级。

## AI 报告上下文

如果报告 prompt 需要加入夹支信息，只传结构化摘要：

```text
原局夹拱：年月夹丑，藏己癸辛，对日主为正财/正印/正官，拱天乙贵人。
```

报告表达应遵守：

- 可提示“意料之外、情理之中”的暗线。
- 可结合十神解释暗藏资源、关系、压力、贵人等。
- 女命夹官杀可提示暗线关系或意外对象，但避免绝对断语。
- 不说夹支改变原局五行、用神或命格。

## 测试口径

后端单元测试：

- 相邻同干隔位生成夹支。
- 反向地支顺序同样生成同一夹支。
- 天干不同不生成。
- 地支不隔位不生成。
- 年柱-日柱、年柱-时柱、月柱-时柱不生成。
- 大运、流年不参与生成夹支。
- 夹支藏干和十神按日主正确计算。
- 夹支神煞只按地支引用，不构造虚柱天干。
- 大运或流年冲、合、刑、害夹支输出中性事件信号。
- 夹支参与补足三合、三会输出中性事件信号。

前端测试：

- 有 `gong_jia` 时展示“原局夹拱”模块。
- 无 `gong_jia` 或旧快照缺字段时不展示且不报错。
- 模块不渲染为第五柱。
- 神煞徽章文案体现“拱神煞”。

## 风险与控制

| 风险 | 控制 |
| --- | --- |
| 虚支被误认为第五柱 | 字段命名、文案和 UI 都强调“虚支/暗藏信号” |
| 夹支影响既有核心结论 | 第一版不接入五行、用神、命格、身强弱 |
| 事件信号过度判吉凶 | 第一版统一中性 |
| 神煞误算为虚柱神煞 | 只计算可按地支引用的神煞 |
| 老命盘快照兼容问题 | `gong_jia` 可选，缺失时前端降级 |

## 实施顺序建议

1. 后端新增夹支数据结构与检测函数。
2. 在 `Calculate` 中生成 `gong_jia`。
3. 为夹支补齐藏干、十神、地支型神煞。
4. 将夹支接入大运/流年事件引动，输出中性信号。
5. 前端新增“原局夹拱”展示模块。
6. 将夹支摘要接入 AI 报告上下文。
7. 补齐后端和前端测试。
