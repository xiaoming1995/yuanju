# 合盘结果页展示优化设计

## 背景

当前合盘结果页已经能展示四维分数、结构化证据和 AI 解读，但页面阅读顺序仍偏“算法输出”。用户进入结果页后，能先看到结论和证据，却看不到双方命盘本体，缺少“这份合盘到底是根据哪两个人、哪组八字算出来的”这一层上下文。

这会带来两个问题：

- 用户难以快速确认结果是否对应当前对象
- 证据和结论显得悬空，不利于建立信任感

## 目标

- 让结果页先展示结论，再展示双方命盘摘要，再展示依据
- 把双方四柱八字明确展示出来
- 在不做完整单人命盘页复刻的前提下，补足用户最关心的命盘信息

## 信息层级

结果页调整为以下顺序：

1. 总评 Hero
2. 双方命盘摘要
3. 四维评分
4. 关键证据
5. AI 合盘解读

## 方案

### 1. 总评 Hero

顶部保留合盘关系主标题，但强化结果导向：

- `我 × 对方`
- 契合等级文案
- summary tags
- 一句短结论

这一块负责让用户先知道“结论是什么”。

### 2. 双方命盘摘要

新增双栏摘要卡，分别展示双方：

- 身份名
- 性别
- 出生年月日时
- 四柱八字
- 日主
- 五行概览

这里不展开十神、神煞、调候等完整命盘细节，只保留足够建立上下文的核心信息。

### 3. 四维评分

保留现有四维分数：

- 吸引力
- 稳定度
- 沟通协同
- 现实磨合

这一层放在双方命盘摘要之后，让用户先知道“命盘是什么”，再看评分。

### 4. 关键证据

保留现有结构化证据，但做两点优化：

- 继续使用中文化标签，不暴露内部英文枚举
- 作为“依据层”而不是“首屏主信息”

### 5. AI 合盘解读

继续放在结果页末段，作为结构化结果之上的自然语言总结。

## 数据来源

优先复用现有 `participants[].chart_snapshot`：

- 从 snapshot 里读取四柱、日主、五行等展示字段
- 不新增后端接口
- 不改合盘算法

## 边界

本次不包含：

- 结果页重做为完整专业双命盘对照表
- 新增复杂图表或雷达图
- 后端返回结构变更
- 合盘评分逻辑调整

## 验证

- 结果页顶部可看到明确总评
- 用户可直接看到双方四柱八字
- 四维评分和关键证据仍正常显示
- AI 合盘解读不受影响
