# 用神算法 10 条审计

## 用户提供的命理算法（权威规范）

> 调候为急、扶抑优先
>
> 如果原局至寒至热，则火水调候为最急。
> 如果原局冷热适度，日主过强则抑制优先；过弱，则生扶优先。
> 这两个排列优先考虑完了之后，再考虑格局配合、通关、用药。
>
> 用神直接关系通过原局断准命主人生层次；关系大运吉凶悔吝。
> 如果原局用神得力不受刑冲，则终身顺利，反之终身发展受阻。
> 如果大运为用神或生助用神（即为用神原神），则此轮大运为好运。
> 如果大运为忌神或冲克用神，则此轮大运为霉运。
> 除此之外，都是平常运。
>
> 考察大运、流年与原局之间的刑冲克合穿破，是看应期的关键。
> 原局用神被刑冲克合穿破，果断判凶。
> 原局忌神被刑冲克合穿破，反而化吉。

## 10 条审计项

### A 组：用神判定主轴（审计 1-3）

#### ❌ 1. "调候为急" → 至寒至热 → 火水调候

- **代码位置**：grep "至寒/至热/寒燥/过寒/过热" 全 `pkg/bazi/` 无结果
- **现状**：调候字典命中即拍板，**不论冷热**直接采纳为综合用神
- **断点函数**：`inferYongshenWithTiaohouPriority` (`yongshen.go:328`)
- **测试盘表现**：寅月乙木丙透两位（已暖），非至寒，但仍走调候字典路径

#### ❌ 2. "扶抑优先" → 冷热适度 → 身强抑、身弱扶

- **代码位置**：扶抑函数存在但被旁路
  - `calcFuyiStrength` (`yongshen.go:196`) — 位置规则
  - `calcWeightedYongshen` (`engine.go:538`) — 月令加权
  - `dayMasterStrengthLevel` (`event_signals.go:1091`) — 5 档评分
- **现状**：调候命中后不调用任何扶抑函数；同时上述 3 个身强弱算法**互相矛盾**：
  - `calcFuyiStrength` 判测试盘 = 身弱（克泄数 6 ≥ 3）
  - `calcWeightedYongshen` 判测试盘 = 身强（helpPct 60% > 40%）
- **断点函数**：`inferYongshenWithTiaohouPriority` 优先级倒置

#### ❌ 3. "再考虑" → 格局配合 / 通关 / 用药

- **格局**：`mingge.go` 存在，**未介入**用神判定
- **通关**：仅在 `collectYingqiSignals` 局部使用（"克但流通"降为中性，`event_signals.go:594`）
- **用药**：完全无实现
- **现状**：用神判定层完全没有这三层

## 测试盘对账 (1996-02-08 20:00 男, chart 221c84e7)

**四柱**: 丙子 庚寅 乙亥 丙戌
**日主**: 乙木

### 用户算法判定路径
1. 寒热判断：丙火两透 + 寅月木旺 → 非至寒，非至热 → 跳过调候
2. 身强弱：建禄 + 亥水印 + 子水印 → 身强偏旺
3. 扶抑取用：克泄耗 → **用神 = 火/土/金**
4. 调候叠加：寅月乙木需丙暖（丙已透）→ 强化"火"为用主
5. 综合：**用神=丙火（主）+ 土金（辅），忌神=水木**

### 代码实际产出
- `bazi_charts.yongshen` = "火水"  ← 错
- `bazi_charts.jishen` = "木土"     ← 错
- 日支亥水 → 被标"用神位"            ← 错
- 巳火（大运地支）→ 冲亥（"用神位"）  → AI 写"巳火忌神发力"
- 癸水（大运天干）→ 属"用神"五行集    → AI 写"癸水为调候喜神"

癸巳大运段 AI 总结同时出现：
- "癸水为调候喜神"（基于错标签）
- "巳火忌神发力"（基于错位置标签）
