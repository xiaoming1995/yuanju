## ADDED Requirements

### Requirement: 命理解读动态加载状态展示
系统在此过程中必须隐藏原本的骨架屏，转而显示具有阶段性提示的动态进度 UI，直到后端拉取完整报告。

#### Scenario: 点击生成 AI 解读按钮
- **WHEN** 用户处于命盘结果页，且未生成 AI 报告时，点击「生成 AI 命理解读」按钮
- **THEN** 按钮和原始空白引导状态消失，UI 切换为大模型推演进度展示态
- **THEN** 系统每隔 3-5 秒切换一句带有专业命理色彩的推演文案（如「提取四柱特质...」、「推演子平格局...」等），直到最后一步定格为「正在精心排版专属报告...」
- **THEN** 显示动效必须与整体深色高质感主题保持风格一致，例如带有呼吸光的文本或平滑的淡入淡出动画。

### Requirement: 继续兼容非流式等待情况
在前端未接入 SSE 流式传输的前提下，加载状态的持续时间不可预测（通常大于 15 秒），进度步骤需确保能在漫长等待期无缝衔接。

#### Scenario: 加载时间过长时
- **WHEN** 动态文案步骤轮播到最后一个步骤时（AI API 尚未返回结果）
- **THEN** 系统停留于最后一个步骤，并且显示如脉冲或呼吸等的局部 CSS 动画，避免用户以为系统假死。
