## Context

当前缘聚平台在生成八字命理解读时，使用的是基于深色主题和亮金色主色的设计语言。在发起 AI 生成请求后（通常需要耗时十数秒至上百秒），原有的「✨ 生成 AI 命理解读」主 CTA 按钮会被一套简单的深灰色骨架屏（Skeleton）所替代。这在视觉上极其扁平暗淡，无法给与用户明确的「正在高负荷推演中」的积极预期，反而容易让用户以为页面停滞。

为了缓解生成的等待焦虑、提升专业感，我们采取「沉浸式动态进度指示器（大模型沉思交互）」的方案来替换原本静态的倒计时或纯转圈状态。

## Goals / Non-Goals

**Goals:**
- 提供从按钮点击到大段生成结果展现之间的平滑视觉过渡。
- 采用动态文本步骤（模拟大模型执行四柱抽取、格局判定、调候抓取等内部思维过程）强化系统的命理专业认知度。
- 全程通过 React 状态管理，组件内聚化处理动画，无需改动现有后端逻辑。

**Non-Goals:**
- 不涉及改变现在的非流式（non-streaming）全量等待策略（流式响应为独立迭代计划）。
- 不涉及对实际底层服务状态的长连接轮询（前端只需展现一套极具安抚效果的伪进度步骤轮播即可）。

## Decisions

- **组件化隔离设计：** 
  分离出一个新的 `AILoadingIndicator` 或类似的内部渲染逻辑，取代长而零散的 skeleton 代码。
- **阶段性文字动画（核心交互）：** 
  配置一个 4 到 5 步的动态推演文案数组（例如："提取四柱十神..." -> "推演子平格局..." -> "查阅穷通鉴查调候..." -> "综合撰写专属报告..."）。
  利用 `useEffect` 与 `setInterval/setTimeout`，每隔几秒推进展示下一步，直至 AI 请求完成，给漫长的等待期填充实际内容感。

## Risks / Trade-offs

- **Risk**: 进度条轮播可能走完所有文案但请求仍未成功。
- **Mitigation**: 最后一个步骤的文案应设定为一个无限持继态（例如："【最终生成】AI 正在精心排版你的深度命理报告..."），避免步骤走完后的冷场，并结合跳动的 "..." 或 CSS loading 呼吸灯动画持续吸引注意力。
