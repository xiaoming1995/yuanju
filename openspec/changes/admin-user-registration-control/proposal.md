## Why

当前普通用户注册入口完全开放，平台无法在内测、邀约制运营或灰度阶段控制新用户进入。同时 Admin 后台只能查看用户列表，不能为线下邀约、企业客户或测试账号主动创建普通用户账号。

## What Changes

- 新增 Admin 创建普通用户能力：管理员可在后台输入邮箱、昵称和初始密码创建普通用户账号。
- 新增公开注册开关：管理员可启用或关闭前台公开注册入口。
- 公开注册关闭时，普通用户 `POST /api/auth/register` SHALL 被后端拒绝；Admin 创建用户仍可用。
- 前端注册入口根据注册开关调整展示：关闭时隐藏或弱化注册 CTA，并在注册页展示“暂未开放公开注册”的明确提示。
- 用户列表补充创建入口，并刷新列表展示新建用户。
- 初始密码由管理员设置，密码规则沿用普通注册的最小长度要求。

## Capabilities

### New Capabilities

- `admin-user-registration-control`: Admin 创建普通用户账号与公开注册入口启停控制。

### Modified Capabilities

- None.

## Impact

- **后端**：新增注册设置读写接口、Admin 创建普通用户接口；调整普通注册服务在公开注册关闭时返回拒绝错误。
- **前端**：Admin 用户页面新增创建用户表单和公开注册开关；普通注册页、导航注册入口和结果页注册 CTA 根据开关状态调整。
- **数据库**：新增系统设置表或同等持久化机制保存 `registration_enabled`；可选为 `users` 增加来源字段用于区分自注册与 Admin 创建。
- **安全与兼容**：注册入口控制必须由后端强制执行；默认保持公开注册开启，避免升级后破坏现有站点行为。
