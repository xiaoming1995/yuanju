## Context

缘聚八字命盘当前展示「大运 → 流年」两层时间维度。`lunar-go v1.4.6` 已原生提供 `LiuNian.GetLiuYue()` 返回 12 个流月对象，每个流月可取干支（五虎遁算法）、旬空等信息；节气起止日期可通过 `Lunar.GetJieQiTable()` 查表获得。后端流月计算能力完备，只需暴露接口并构建前端 UI 即可。

## Goals / Non-Goals

**Goals:**
- 新增 `POST /api/bazi/liu-yue` 公开接口，传入流年公历年 + 日主天干，返回 12 个流月数据（干支、天干十神、地支十神、节气名、节气起止日期、当前流月 index）
- 前端流年格点击弹出流月抽屉（LiuYueDrawer），展示 12 月网格并高亮当前流月
- 抽屉内提供年份切换器（默认当前年），支持查询任意年份流月

**Non-Goals:**
- 不修改现有 `/api/bazi/calculate` 响应结构（流月数据按需请求，不预置）
- 不在流月层级加入 AI 解读
- 不支持流日/流时粒度

## Decisions

### 1. 流月干支计算：复用 lunar-go 五虎遁

**决策**：直接调用 `LiuNian.GetLiuYue()[i].GetGanZhi()`，不自研五虎遁算法。

**理由**：`lunar-go` 已实现完整的五虎遁（甲己年丙作首...），经过验证，自研重复造轮子且易出误差。

**替代方案**：硬编码查表 → 否决，不如库的精确度高。

---

### 2. 节气起止日期：服务端查表返回

**决策**：后端通过 `calendar.NewSolar(year,1,1,0,0,0).GetLunar().GetJieQiTable()` 获取当年所有节气的精确公历日期，与 12 个流月一一配对后返回给前端。

映射关系（流月 index → 节气名）：
```
0=寅月→立春, 1=卯月→惊蛰, 2=辰月→清明, 3=巳月→立夏
4=午月→芒种, 5=未月→小暑, 6=申月→立秋, 7=酉月→白露
8=戌月→寒露, 9=亥月→立冬, 10=子月→大雪, 11=丑月→小寒
```

起止日期：第 i 月的开始 = 第 i 个节气日期，结束 = 第 i+1 个节气日期 - 1天（丑月结束为次年立春前一天）。

**理由**：节气日期年份不同时略有差异（通常差 1-2 天），不能硬编码，必须动态计算。

**替代方案**：前端硬编码节气日期 → 否决，跨年份会不准。

---

### 3. 当前流月 index 的判断：服务端计算

**决策**：服务端接收请求时获取当天日期，与当年节气表对照，找到刚过去的奇数位节气（立春/惊蛰/清明...），映射出 `current_month_index`（0-11），随流月数据一起返回。

**理由**：前端若自行判断需要节气数据，增加复杂度；服务端一次计算更简洁。

**替代方案**：前端基于当月阳历推算 → 否决，节气和阳历月份不对齐（如 2026 年 4 月 4 日仍在卯月，4 月 5 日才入辰月）。

---

### 4. 接口设计：POST + JSON Body，无需鉴权

**决策**：
```
POST /api/bazi/liu-yue
Body: { "liu_nian_year": 2026, "day_gan": "甲" }
Response: {
  "liu_yue": [
    {
      "index": 0,
      "month_name": "寅月",
      "gan_zhi": "丙寅",
      "gan_shishen": "食神",
      "zhi_shishen": "偏印",
      "jie_qi_name": "立春",
      "start_date": "2026-02-04",
      "end_date": "2026-03-05"
    },
    ...
  ],
  "current_month_index": 2
}
```

**理由**：流月计算是纯算法，无需用户身份；与 `/calculate` 风格一致（公开接口）。

---

### 5. 前端交互：Drawer（抽屉）组件，非内联展开

**决策**：点击流年格弹出右侧抽屉，抽屉内展示 12 个流月网格 + 年份切换器。

**理由**：流月数据量较大（12 格），内联展开会使大运列区域过于复杂；抽屉可给足够空间展示节气日期等详细信息，且不破坏大运时间线的横向扫视体验。

**替代方案**：内联下方展开 → 否决，层级叠加后页面信息密度过高。

---

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|---------|
| 跨年丑月的结束日期计算（小寒跨年） | 丑月结束日期 = 取次年年份的节气表中立春日期 - 1天，后端统一处理 |
| 年份切换时的网络延迟 | 前端显示 loading 状态；流月计算轻量（< 5ms），延迟可忽略 |
| 流年干支歧义（农历年 vs 节气年） | 明确使用节气年（立春为界），与 lunar-go 一致 |
| 手机端抽屉宽度不足 | LiuYueDrawer 在移动端改为底部全屏 sheet 展示 |
