## Context

当前系统计算的大运（Dayun）包含了各柱干支及其对日主的十神映射，但未能从经典命学算法层面为其赋予明确的吉凶评级。目前前端仅仅展示客观干支数据，吉凶推断全依赖于生成长文报告时 AI 的泛化理解。《金不换大运歌》能依据日干与月支，直接针对未来运势的行走方向（比如南方火运、西方金运）给出权威的结论。

## Goals / Non-Goals

**Goals:**
- 在 `BaziResult` 的 `Dayun` 解析中自动解析大运地支的五行方位（东、南、西、北）。
- 引入静态字典 `jin_bu_huan_dict.go` 支持 120 种日干+月支的组合校验。
- 在基础引擎中直接附带大运吉凶标签和原断语。
- 前端 `DayunTimeline` 支持渲染这些标签直观指示运势高低。

**Non-Goals:**
- 处理神煞流派的复杂刑冲合害对《金不换》原诀的细节干预（保持大方向查表为主）。

## Decisions

1. **方位推算是通过地支三会局**：
   - 寅卯辰 → 东方木
   - 巳午未 → 南方火
   - 申酉戌 → 西方金
   - 亥子丑 → 北方水
   使用一个单独的公用函数，把每个大运地支转化成对应的运势方向。
   
2. **打分机制（Eval）设计**：
   为方便前端根据吉凶着色，设计标准的返回结构：
   ```go
   type JinBuHuanResult struct {
       Level   string // 比如 "吉", "凶", "平"
       Keyword string // 比如 "发财", "剥丧"
       Text    string // 原文解释
   }
   ```
   给每一柱大运（`DayunItem`）增加 `JinBuHuan *JinBuHuanResult` 字段。

## Risks / Trade-offs

- **Risk**: 字典构建工作量较大（120条规则）。
  **Mitigation**: 本提案实现时，先注入数据结构框架，并完整填充核心常用规则（如甲乙木）；前端设计保证对于部分没有匹配字典返回 NULL 的情况优雅降级（不展示）。
- **Risk**: 金不换判定与其他系统（如神煞或现代调候）部分理论可能会产生表象冲突。
  **Mitigation**: 前端注明此评级来源于《金不换大运歌》流派断语，供古籍爱好者和算法派同时参考。
