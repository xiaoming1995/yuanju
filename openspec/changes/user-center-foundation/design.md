## Context

当前普通用户登录后可以查看历史记录、生成报告、查看合盘记录，但缺少一个统一的个人中心。用户模型只有 `id/email/nickname/created_at`，认证接口只有注册、登录和 `/api/auth/me`。前端顶部导航仅展示昵称和退出按钮，底部导航登录后把第三个入口指向历史页。

后续充值系统、点数流水、PDF 模板个性化都需要稳定的账户入口。本变更先建立个人中心基础，不引入支付渠道和模板引擎。

## Goals / Non-Goals

**Goals:**

- 新增 `/profile` 个人中心页面，作为登录用户的“我的”入口。
- 新增鉴权聚合接口，返回用户基本信息、账户统计、最近记录和后续能力入口状态。
- 在个人中心中提供历史记录、合盘记录、PDF 模板设置、充值/点数等入口位置。
- 保持历史页、结果页、合盘页的现有功能不变。
- 为后续充值系统和 PDF 模板定制预留清晰的数据结构和页面入口。

**Non-Goals:**

- 不接入微信支付、支付宝或其它真实支付渠道。
- 不实现点数账户、扣费、订单、回调或退款。
- 不实现 PDF 模板选择、模板保存或付费模板权限。
- 不修改 JWT 结构。
- 不要求迁移历史数据。

## Decisions

### D1：新增 `/api/user/profile` 聚合接口

接口由普通用户 JWT 鉴权保护，返回：

- `user`: 当前用户基本信息。
- `stats`: 命盘数量、AI 报告数量、合盘记录数量。
- `recent_charts`: 最近若干命盘摘要。
- `recent_compatibility`: 最近若干合盘摘要。
- `features`: 后续模块入口状态，例如 `wallet`, `pdf_templates`，第一版返回 `coming_soon` 或 `disabled`。

**理由**：个人中心首屏需要多个模块的数据。用后端聚合可以避免前端并发请求散落，也方便后续把钱包余额、会员状态、模板设置追加进同一个账户概览。

**替代方案**：前端复用 `/api/auth/me`、`/api/bazi/history`、`/api/compatibility/readings` 拼装。这个方案短期少写后端，但个人中心会快速变成多个接口耦合点，后续接钱包和模板设置时会更乱。

### D2：新增 `ProfilePage`，导航从“历史”升级为“我的”

前端新增 `ProfilePage` 和 `/profile` 路由。顶部登录用户昵称可进入个人中心；底部导航登录后第三个入口指向 `/profile` 并显示“我的”。历史记录作为个人中心中的一个主入口，也保留 `/history` 独立页面。

**理由**：个人中心是账户级入口，历史只是其中一个功能。这样后续充值和 PDF 模板不会挤到历史页。

### D3：个人中心第一版以账户工作台为主

页面结构建议：

- 账户摘要：昵称、邮箱、注册时间。
- 数据卡片：命盘数、报告数、合盘数。
- 快捷入口：历史记录、合盘记录、继续测算。
- 后续能力：充值/点数、PDF 模板设置，以“即将开放”或“敬请期待”状态呈现。

**理由**：这符合当前产品阶段。先让用户知道“我的账户”在哪里，再逐步接入商业化能力。

### D4：不新增数据库表

本变更只聚合已有表数据，不新增钱包、订单、模板设置表。充值和 PDF 个性化在后续变更中单独建表。

**理由**：避免在没有支付产品规则和模板权限规则时提前设计错误表结构。个人中心基础版不需要新表也能形成稳定入口。

## Risks / Trade-offs

- **[入口看起来有充值但不能用]** → 后续能力必须明确显示“即将开放”，不可误导用户完成付费动作。
- **[统计查询性能]** → 第一版用简单 COUNT 聚合即可；后续数据量大时再加缓存或物化统计。
- **[个人中心变成杂乱入口页]** → 只放账户、记录、后续商业化入口，不把测算结果详情搬进个人中心。
- **[后续支付设计被基础接口限制]** → `features` 只表达入口状态，不定义钱包余额或订单模型，避免提前绑定。

## Migration Plan

1. 新增后端 profile 聚合接口和相关查询方法。
2. 新增前端 API 类型和 `ProfilePage`。
3. 更新路由与导航入口。
4. 验证登录用户能打开个人中心，未登录用户被引导登录。
5. 验证历史页、合盘历史页、结果页仍保持现有行为。

## Open Questions

- 充值系统后续采用“点数包”还是“会员订阅”需要在单独变更中确定。
- PDF 模板个性化第一版是否允许用户自定义署名/封面标题，也留到单独变更中确定。
