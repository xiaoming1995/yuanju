## Context

八字命盘的核心输出——喜用神（`yongshen`）与忌神（`jishen`）——已经以结构化字段的形式稳定存在于后端 API 响应中。然而前端仅将其作为两个纯文本标签展示，没有任何视觉延伸。

命理学中，五行本就自带一套完整的符号体系（颜色、方位、数字、形状），这是几千年积累的"设计语言"。利用这套现成规则，我们可以在不增加任何后端工作量的前提下，为用户创造强烈的「个性化归属感」。

## Goals / Non-Goals

**Goals:**
- 建立一套完整的「五行视觉映射规则集」作为所有视觉化功能的底层数据源
- 在命盘结果页为用户展示基于喜用神的个性化「属性徽章」（颜色、方位、数字、幸运色块）
- 使用纯前端 SVG 程序化生成「命理头像」，无需外部 API，零成本运行
- 支持用户下载/保存自己的命理头像

**Non-Goals:**
- 不接入外部 AI 生图 API（DALL-E / 即梦等），本期不做
- 不涉及后端任何改动
- 不做用户头像持久化存储（生成后仅提供客户端下载，不上传服务器）
- 不支持用户自定义头像风格参数

## Decisions

### 1. 五行映射规则集：静态 TypeScript 常量

五行视觉规则是固定的命理常识，无需动态计算，使用一个 `WUXING_MAP` 常量对象即可：

```typescript
// src/lib/wuxingColorSystem.ts
export const WUXING_MAP = {
  木: {
    color: '#4CAF7D',      // 翠绿
    darkColor: '#2E7D52',
    shape: 'tall',         // 上升/细长
    direction: '东方',
    season: '春',
    luckyNumbers: [3, 8],
    luckyMaterial: '木',
    emoji: '🌿',
  },
  火: { color: '#FF6B35', shape: 'sharp', direction: '南方', ... },
  土: { color: '#C8962E', shape: 'square', direction: '中央', ... },
  金: { color: '#A0A8B0', shape: 'round', direction: '西方', ... },
  水: { color: '#4A7FA5', shape: 'wave', direction: '北方', ... },
}
```

**为何选静态常量而非配置表**：五行规则是不变的，无需数据库存储；TypeScript 类型安全，IDE 自动补全。

### 2. 喜用神徽章组件：YongshenBadge

将 `yongshen` / `jishen` 字符串拆解为单个五行字符（支持"木火"、"木"等多种格式），查询 `WUXING_MAP` 后渲染。展示内容：

```
┌──────────────────────────────────────┐
│  ✦ 命元特质                           │
│                                      │
│  喜  [🌿木] [🔥火]  →  颜色色块示意  │
│  忌  [⚪金] [💧水]  →  颜色色块示意  │
│                                      │
│  幸运方位  东方 · 南方                │
│  幸运数字  3 · 8 · 2 · 7             │
│  幸运季节  春 · 夏                    │
└──────────────────────────────────────┘
```

### 3. 程序化 SVG 头像：MingpanAvatar

核心算法：
- 以喜用神第一五行为**主色**，第二五行（如有）为**辅色**
- 以忌神第一五行为**对比色/边框色**
- 生成一个 320×320 的 SVG，分三层：

```
Layer 1: 背景渐变（主色 → 辅色）
Layer 2: 主纹样（根据五行形状规则生成几何图案）
          木 → 竖纹/竹节   火 → 三角射线
          土 → 方格        金 → 同心圆    水 → 波浪
Layer 3: 边框装饰（忌神对比色，细描边）
Layer 4: 中央字符（日主天干汉字，白色/金色大字）
```

头像输出为 SVG 字符串，可直接嵌入页面，也可通过 `SVGtoCanvas → canvas.toBlob()` 触发浏览器文件下载（PNG格式）。

**为何选程序化而非 AI 生图**：
- 零 API 成本 vs ¥0.05~¥0.2/张
- 响应即时（< 1ms）vs 3~10秒生成等待
- 结果视觉上可解释（"你的头像是绿色因为你喜木"）
- 版权清晰，无内容审核风险

### 4. 集成位置

- **命盘结果页**（已登录/未登录均可见）：在五行饼图下方新增 `YongshenBadge` 区块
- **头像生成**：在 `YongshenBadge` 下方放置 `MingpanAvatar` 预览 + 「下载头像」按钮
- **状态处理**：`yongshen` 为空时（AI 报告未生成前）显示骨架屏占位

## Risks / Trade-offs

- **Risk: SVG 在不同浏览器的 Canvas 渲染差异** → 使用 `foreignObject` 替代方案或直接提供 SVG 下载；Canvas 下载作为增强功能，降级为 SVG 下载
- **Risk: yongshen 字段包含非五行字符（如"无"、""）** → 在解析函数中过滤，只取"木火土金水"字符，其他字符跳过
- **Risk: 喜用神为空时组件空状态丑** → 设计"待推算中"骨架屏 + 提示文案
