## ADDED Requirements

### Requirement: AI 八字报告生成
系统 SHALL 将八字计算结果构造为结构化 Prompt，调用大模型 API 生成分段式自然语言解读报告。

#### Scenario: 报告正常生成
- **WHEN** 八字计算完成且用户已登录，点击"生成AI解读"
- **THEN** 系统在30秒内返回包含【性格特质】【感情运势】【事业财运】【健康提示】四个章节的报告

#### Scenario: 报告内容语言风格
- **WHEN** AI 报告生成完成
- **THEN** 报告使用通俗中文，避免晦涩术语，每章节200-400字，读者无需命理基础也能理解

#### Scenario: 大模型 API 超时处理
- **WHEN** 大模型 API 响应超过30秒
- **THEN** 系统返回提示"报告生成超时，请稍后重试"，不保存空报告

---

### Requirement: 报告缓存复用
系统 SHALL 对同一命盘的 AI 报告进行缓存，避免重复调用大模型 API。

#### Scenario: 复用已有报告
- **WHEN** 用户对同一生辰数据再次请求 AI 报告
- **THEN** 系统直接返回已缓存的报告，不重复调用大模型

---

### Requirement: 报告免责声明
系统 SHALL 在每份 AI 报告底部显示固定免责声明。

#### Scenario: 声明展示
- **WHEN** AI 报告渲染完成
- **THEN** 报告末尾显示"本报告由 AI 辅助生成，内容仅供参考，不构成任何决策建议"
